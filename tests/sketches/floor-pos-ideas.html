Five ways to work the POS straight from the floor

The floor already knows who's where (the TableSession spine). The missing half is acting on the check in place — add a round, fire a course, split, take payment — without walking to a station. Here are five interaction models, from a light refinement of what's shipped to a keyboard-first power tool. Each writes to the same PosTab engine, so they compose rather than compete.

Reuses the CorePos / PosTab engine Floor-first, no context switch Pick one primary + steal ideas from the rest
Where we are today. Tapping a table already opens an embedded CorePos as a docked overlay (core-check-panel), and seating a booking now auto-opens a dine-in tab on the table. So idea 1 is a refinement of the real baseline; ideas 2–5 layer faster or denser ways to reach the same actions.
1

Docked check drawer

→ the safe default · refine what's shipped

Tap a table and its full check slides in as a right-hand drawer over the floor — the whole till (build · course · modify · split · discount · pay) without navigating away. The floor stays visible and dimmed behind it, so the server keeps room awareness while working one check.

2

Long-press quick-add radial

→ the speed play · "another round" in one tap

Press-and-hold a table and a radial blooms the most likely next items for that check — drawn from what they've already ordered, the upsell engine, and time-of-meal (drinks early, dessert + espresso late). One tap fires an item onto the tab without ever opening the full till. The 80% case — "another Aperol on T6" — becomes a half-second gesture.

3

Split-screen floor + till station

→ the workhorse · a fixed POS terminal

A two-pane mode for a dedicated station or a wall tablet: the floor map lives on the left, the selected table's live till on the right. Tapping any table swaps the right pane instantly — no open/close, no overlay. The bartender or a POS-anchored server flows table-to-table without a single modal.

4

Expand-in-place tile till

→ the spatial play · the table IS the check

Tap a table and the tile itself expands in place into a mini-till — the floor grid reflows around it. Current items, running total, +add, fire, pay, all inside the table's own footprint. Nothing overlays; the check literally lives where the table is on the map, so spatial memory ("the check in the corner") does the navigating.

5

Command bar — type or speak any table

→ the power tool · operate without pointing

A command palette (the ⌘K bar the shell already has) that operates any check by typing: T6 add 2 aperol, T6 pay card, T11 fire mains. Fuzzy-matched items, one-line to fire. Pair it with voice for a hands-full server ("table six, two aperol"). It doesn't replace touch — it's the fast lane for anyone who'd rather not hunt-and-tap.

Recommendation

Harden idea 1 as the floor of capability, then layer idea 2 for speed. The docked drawer is already mostly built and covers every action safely — polish it (swipe-dismiss, keep-open-on-switch, peek width) so nothing is ever unreachable. Then add the long-press quick-add radial on top: it turns the 80% case ("another round", "fire dessert") into a one-tap gesture without replacing the drawer for the long tail. That pairing — a complete fallback plus a fast path — gives the biggest lift for the least risk.

Offer idea 3 (split station) as a per-device mode for a mounted bar/pass tablet — it's the same CorePos in a rail, so it's cheap once the drawer exists. Treat 4 and 5 as opinionated follow-ons: expand-in-place if the floor goes fully touch-native, the command bar once there's appetite for a power-user / voice lane. All five write to one PosTab truth, so they stack — you're choosing sequence, not making an exclusive bet.
