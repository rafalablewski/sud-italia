Core · loading skeletons

The client-fetched Guest & Service surfaces (CRM · Loyalty · Inbox · Slots · Book) fetch their rows in a useEffect, so their first frame has no data. We replaced the dead centered "Loading…" label with a shimmer skeleton shaped like the content arriving — the cheapest way to make a fetch read as fast instead of stalled. One shared primitive (src/core/shared/Skeleton.tsx) drives the whole suite.

Surfaces touched: CRM, Loyalty (roster · wallets · redemptions), Inbox (list · transcript), Slots, Book — 8 bare pane-msg "Loading…" blocks replaced. The shimmer + colours are Core-theme tokens (.core-suite .skel*); the sweep stops under prefers-reduced-motion. This is Core's own idiom — it does not borrow the admin .v2-page-loading pill. Each container carries role="status" / aria-busy.
