core layout system · one ActionBar, every page

1 · Identity

Who am I looking at. Section glyph + leaf name + one muted context line (date · service · location). Replaces the old core-crumb and the oversized core-sectionhead — the nav already says service:book, so this stays slim.

2 · Controls

How I'm viewing it. View / mode / scope / status live in one shared .seg primitive; date & filters are .pills. Everything that changes what's shown (never the data) sits here, always left of the actions.

3 · Actions

What I can do. Secondary actions are ghost buttons; there is always exactly one filled primary, pinned far right. 3+ secondaries collapse behind ⋯. Mutations never mix into the controls slot.

Shared StatStrip

Same six-tile grid on every page. One primitive, one tone system (green ok · blue info · amber watch · red critical). Value, boxed digit, sub-label — nothing per-page except the data.
