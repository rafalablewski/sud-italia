Audit §2.1 – 2.5

Cart Upsell v2 — visual draft

Four new surfaces, all inside the existing cart drawer + add-to-cart flow. Nothing here blocks the happy path — every prompt is one-tap-add, dismissable, and contextual. Same tokens, fonts, icons, and copy patterns as the live drawer (CartDrawer.tsx, CartUpsell.tsx, DeliveryProgress.tsx) — just augmented.

Pick a time-of-day variant, then hit + Add Margherita to see the toast, tap any chip to add it, and use Push to free-delivery to watch the progress bar flip green.

What's new

The four surfaces in this redesign

  • 1 Inline toast on add-to-cart. Seed-only copy: "Customers usually add an espresso." No CTA, no block. Auto-dismisses in 4s. Today the public side is silent on add — only the floating cart badge animates. This is net-new.
  • 2 "Complete the meal" chips, 3-up grid. Replaces the existing vertical list in CartUpsell.tsx. Same Sparkles header + cream-on-gold card skin — just 3 columns of one-tap chips, margin-ranked (espresso first per §2.4). Tap once, the chip flips green, item is in cart.
  • 3 Animated free-delivery progress bar. Today's DeliveryProgress.tsx already has the gradient bar — this adds a continuous shimmer so motion catches the eye, plus a smoother red→green transition when the threshold clears.
  • 4 Time-of-day banner at cart top. Sits above the cart items, same chrome family as ComboDealBanner.tsx. Six hour-window variants; one banner at a time, picked server-side by local hour × customer state. Lunch + Last-call apply a real discount; the rest deep-link.
Out of scope for this sketch

Deferred to follow-up tickets

  • Apple Pay / Google Pay primary CTA at the Stripe sheet — Top-50 #4. Needs Stripe Payment Request API wiring.
  • Item modifiers ("Make it a large +PLN 6") — needs a CartItem.modifiers migration. Top-50 #15.
  • Bundle / decoy engine (Lunch+, Family Feast) — covered under §3.2. The lunch banner here shows the eventual link.
  • Per-customer ML ranking — §9.1. This sketch uses the static margin × attach-rate heuristic from §2.4.
Won't ship (§2.2)

Patterns we will not use

  • No "Are you sure?" confirmation pop-ups.
  • No blocking modal after Add to cart.
  • No guilt copy ("You forgot dessert!").
  • No fake urgency — windows are real, scarcity comes from real inventory.
