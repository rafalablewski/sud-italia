What's new — full §2 coverage

The redesign deltas vs production

  • 1 Inline toast on add-to-cart. §2.1 T+0. Seed-only copy ("Customers usually add an espresso"), auto-dismiss 4s, no CTA, no block.
  • 2 "Complete the meal" 3-up chips above subtotal. §2.1 + §2.4. Margin-ranked (espresso first: 83% GM × 60% attach). One-tap-add, no second screen.
  • 3 Animated free-delivery progress bar. §2.1 + §2.5 Uber Eats. Shimmer overlay + smoother red→green flip. Threshold is per-segment (regulars 49 zł, first-timers 39 zł, Gold free) — note the "tuned for you" annotation next to the target.
  • 4 Apple Pay primary, card secondary. §2.1 T+pay. Stripe Payment Request API wiring lands separately; this mockup shows the intended visual hierarchy.
  • 5 Time-of-day banner — 8 variants. §2.3. Lunch, afternoon, dinner, last-call, pre-order, rain, 14-day lapsed, 3rd-order loyalty unlock. One at a time, picked server-side by hour × weather × customer state.
  • 6 McDonald's default-effect on the lunch combo. §2.5. Combo is pre-applied during the lunch window — the CTA flips to "Remove combo" so non-combo is now the deviant path. Toggle the ToD to see the discount auto-apply.
  • 7 Scarcity strip — real seasonal stock. §2.2 row 7. "Try the seasonal burrata — 7 of 12 left today" uses real inventory from the existing seasonal-items pipeline. Burns down with each add. Tap once to add, tap again to remove.
  • 8 Tier-conditional perk banner. §2.2 row 6. "Make it a Gold-tier order: +pesto bruschetta on the house." Only renders for Gold/Platinum members. Tap once to comp it in, tap again to remove.
Q: admin manageability

Yes — the ToD banners are admin-config, not hardcoded

When this ships, each location's banners live in LocationUpsellConfig.timeWindows[] next to the existing combos array in upsell-settings.json. An admin entry under /admin/upsell exposes:

  • Window — start/end local hour (e.g. 11:30–13:00), days-of-week, location slug.
  • Trigger — hour / weather (rain) / customer state (lapsed, order-#3 unlock).
  • Copy — title, sub-line, badge text, CTA label, icon glyph.
  • Action — apply % discount, add an item to cart, deep-link to a bundle page, or just nudge.
  • Default-apply toggle — whether the banner is opt-out (McDonald's effect) or opt-in.
  • Tier-gating — which loyalty tiers see this banner (Bronze / Silver / Gold / Platinum).

A new capability entry under /admin/capabilities ("Time-of-day upsell windows") will register the surface in the same commit that ships the React component, per Rule #9.

Genuinely out of scope

What requires backend lift, deferred

  • Bundle / decoy engine (Lunch+ tier, Family Feast) — §3.2 territory, separate sprint. The lunch + dinner banners here show the entry points.
  • Per-customer ML upsell scoring — §9.1. Static margin × attach heuristic stands in until the logistic-regression scorer ships.
  • Hour-of-day modulation of "popular this hour" — §1.2 trigger #2. Needs the per-hour item-popularity rollup.
  • Stripe Payment Request API — Apple Pay / Google Pay buttons above show the visual hierarchy; the actual wallet sheet wiring lands separately.
Won't ship — by brand decision

Audit recommendations we reject

  • Item-size modifiers ("Make it a large +PLN 6", "Double cheese", "Add chicken") — §2.2 row 2 + §2.5 Starbucks size-laddering. Sud Italia serves fixed Neapolitan portions. Sizing up or modifying set recipes runs against the brand's core authenticity value, so this entire upsell pattern is overruled.
  • No "Are you sure?" confirmation pop-ups (§2.2).
  • No blocking modal after Add to cart (§2.2).
  • No guilt copy ("You forgot dessert!") (§2.2).
  • No fake urgency — windows are real, scarcity comes from real inventory.
