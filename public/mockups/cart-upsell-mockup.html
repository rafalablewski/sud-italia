Audit §2.1 – 2.5

Cart Upsell v2 — visual draft

Four new surfaces, all inside the existing cart drawer + add-to-cart flow. Nothing here blocks the happy path — every prompt is one-tap-add, dismissable, and contextual. Pick a time-of-day variant and click + Add Margherita to see the full sequence end-to-end.

What's new

The four surfaces in the redesign

  • 1Inline toast on add-to-cart. Seed-only copy ("Customers usually add an espresso"). No CTA, no block. Auto-dismisses in 4s. — see top of phone after pressing Add.
  • 2"Complete the meal" chips above subtotal. Three tap-to-add chips ranked by margin × attach-rate. Espresso comes first because GM 83% × 60% attach = highest profit/order. One tap, no second screen.
  • 3Animated free-delivery progress bar. Visible only when fulfillment = delivery and below threshold. Shimmer animation + remaining-PLN counter. Flips to green/checkmark when qualified.
  • 4Time-of-day banner at cart top. Lunch combo, afternoon espresso break, family feast, last-call clearance, pre-order-for-lunch, returning-after-14d. One banner at a time, picked server-side by local hour × customer state.
Out of scope for this sketch

Deferred to follow-up sprints

  • Apple Pay / Google Pay primary CTA at the Stripe sheet — shown here as a static black button so you can see the visual hierarchy, but the Stripe Payment Request API wiring is a separate ticket (Top-50 #4).
  • Item modifiers ("Make it a large +PLN 6") — needs a CartItem.modifiers migration; covered by §2.2 row 2 and Top-50 #15.
  • Bundle / decoy engine (Lunch+, Family Feast) — covered separately under §3.2. The lunch banner shows the eventual link.
  • Per-customer ML ranking — §9.1; this sketch uses the static margin × attach-rate heuristic from §2.4.
Won't ship

Patterns we will not use (§2.2)

  • No "Are you sure?" confirmation pop-ups.
  • No blocking modal after Add to cart.
  • No guilt copy ("You forgot dessert!").
  • No fake urgency. The time-of-day banner uses real windows; scarcity claims come from real inventory only.
